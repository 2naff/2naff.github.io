현재 일본의 기본정보기술자 시험(基本情報技術者)을 준비하면서 가장 흥미롭다고 여긴 분야가 바로 __'컴퓨터 네트워크'__ 이다.   
비트(bit)와 같은 연산단위부터 하드웨어 그리고 소프트웨어의 종류와 특징은 많이라고는 자부할 수 없지만 어느정도는 알고 있었기에 공부하더라도 큰 흥미를 느끼진 못했다.   
그러나 컴퓨터 네트워크라는 파트에 들어서면서 무지했던 부분이 채워지고, 그로 인해 과거에 왜 그렇게 설정해야만 했는가에 대한 의문도 풀리는 귀한 배움을 얻을 수 있었다.   
이러한 열정이 식기 전에 내가 배운 내용을 공유하면 누군가에게 보탬이 되지 않을까 싶어서 아래에 정리해보았다.

목차
Lan과 Wan 그리고 IP와 MAC의 차이 


LAN(Local Area Network) : 좁은 범위에서 컴퓨터와 주변기기(프린터 등)에 접속하는 네트워크(근거리 통신망)

이러한 LAN의 도입으로 인해 같은 네트워크의 기기끼리 정보 공유나 하드웨어의 자원 공유, 정보 처리를 분산하여 시간을 단축할 수 있게 되었다.   
LAN을 사용하기 위해서는 먼저, Network Interface Card(속칭 랜카드)가 필요하다. 데스크탑 메인보드에 부착된 랜카드에 케이블을 연결하고, 다른 기기들과 같은 LAN에서 연결시키는 접속장치로의 연결이 필요하다.   
이러한 LAN이 여러 기기를 연결하는 형태로 총 3가지가 존재한다.

1. 버스형(バス型) : 하나의 데이터 전송로에 여러 기기가 부착된 형태
2. 링형(リング型) : A라는 기기는 B와 연결되어있고, B라는 기기는 다시 C로.. F라는 기기는 다시 A와 연결되는 꼬리물기 형태다. 단, 이러한 형태는 기기 하나가 고장나버리게 되면 네트워크 간 통신이 되지 않는다.
3. 스타형(スター型) : 각 기기의 케이블이 '허브' 라는 곳으로 연결되며, 이 허브는 다른 허브와도 연결함으로써 쉽게 LAN 확장이 가능하다.
(firefly image1)

WAN(World Area Network) : 물리적으로 멀리 위치한 컴퓨터와 연결해주는 네트워크(광역 통신망; 인터넷이 WAN에 포함된다)

그럼 WLAN은 무엇일까? Wireless-Local Area Network라고 무선 신호를 통해 일정 공간 내의 기기를 연결하는 네트워크를 의미한다.

프로토콜이란, 통신에 필요한 각종 규칙의 집합체로 보면 되는데 어떤 형식으로 정보를 송신할 것인지, 어떤 수단으로, 어떤 케이블 등으로 데이터를 주고 받을지에 대한 것들이 포함되어있다.
이러한 프로토콜은 단순히 하나로 해결되는 것이 아니라 여러 계층으로 나눠서 그 규칙을 세웠는데 그것이 OSI 기본참조모델(OSI基本参照モデル) 또는 OSI 7 Layer Model이라고 일컫는다.


---
IP와 MAC 주소는 본질적으로 다음과 같은 목적을 지니고 있다
네트워크 상에서 개별 장치를 식별 또는 특정하기 위한 주소

세부적으로 두 주소간의 차이를 살펴보자면 IP 주소는 인터넷 환경 속에서 개별 장치를 특정하기 위한 식별자이며, MAC 주소는 같은 네트워크 속에서 장치 자체를 식별하기 위한 주소에 해당한다.
IP 주소는 서비스 제공자, VPN을 활용하여 변경할 수 있으나, MAC 주소의 경우 제품 생산 과정에서 매겨지는 값으로 극소수의 예외를 제외하고는 변경될 수 없다.

Lan 심화
    - 랜의 접속기능
    - (유선)리피터, 브릿지, 루터, 그리고 허브
        해당 장비들은 OSI 계층을 장비 관점에서 보았을 때, 해당 장비들이 OSI 어느 계층에서 있으며 어떤 역할을 하는지 알아둘 필요가 있다.
        리피터(+리피터 허브) : 1계층 물리부에 사용되며, 전송에 필요한 전기 신호를 증폭해준다.
        브릿지(+스위치 허브) : 2계층 데이터 링크 계층에 사용되며, 여기선 MAC 주소를 식별하여 데이터를 보낼지 말지를 결정하는 필터링 역할을 한다. 즉 MAC 식별에 초점을 두고 있으므로 LAN 내부의 통신 담당을 의미한다.
        라우터 : 3계층 네트워크 층에 사용되며, IP 주소를 식별하여 서로 다른 네트워크 간 경로 선택 및 전달하는 역할을 맡는다. LAN과 LAN, LAN과 WAN의 연결에도 사용된다. 
        허브 : 4~7계층에서 사용되며, 서로 다른 프로토콜이나 데이터 형식을 변환하여 상호연동 될 수 있도록 한다.
    - (유선)데이터 통신 방식, 이더넷
        데이터를 보내야 하는데 모두가 동시에 랜을 통해 데이터를 보낼 수는 없다. 이를 컨트롤 하기 위해서는 유선 랜에서 총 두 가지의 접근 제어 방식이 있다.
        1. CSMA/CD 제어 : 전송로가 비어있는지 조사하고 비어있다면 데이터를 보내고, 비어있지 않다면 랜덤한 시간만큼 대기 후 다시 조사하는 방법이다. 이 경우, 충돌이 일어나지 않을 수 있을 거라 생각하지만, 다수의 컴퓨터에서 동시다발적으로 비어있는 걸 확인하고 데이터를 보내면 데이터 충돌이 일어나므로 전송에 필요한 시간이 많이 걸린다.
        2. 토큰 패싱 : 허브나 버스형에서 쓰이는 방식으로 토큰이 전송로를 순환하는데, 이 토큰을 가진 기기가 데이터를 보낼 수 있는 권한을 가지게 된다. 토큰은 하나 밖에 없으므로 충돌이 일어날 수 없다.
